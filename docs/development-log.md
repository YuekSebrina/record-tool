# 开发日志

## 2026-07-22：Phase 1 通用记录原型

基线提交：`177f944 feat: add Phase 1 product prototype`

### 产品方向调整

- 将产品从以书籍、动漫、电影、韩剧为中心的书影音工具，调整为简单的通用记录工具。
- 将记录分为“随记、待办、书影音”三种一级类型。
- 书影音保留为可选模板，不再主导首页信息结构。
- 小程序品牌名称统一为“拾羽”。

### 首页

- 打开首页后可直接输入并保存，不强制进入新页面。
- 随记支持多行文本。
- 待办支持在列表中直接切换完成状态。
- 书影音支持分类和状态选择。
- 动漫、韩剧条目支持圆形 `+1` 快捷操作。
- 最近记录固定显示最新 5 条。
- 最近记录使用缩进、低对比度分隔线。
- 点击最近记录先显示预览弹窗，点击“查看详情”后才进入详情页。
- 白色录入框根据 5 条记录和底部导航的实际位置动态计算最小高度。
- 行内进度更新只刷新数据，不重新测量页面，避免录入框闪烁。

### 全部记录

- 支持正文与备注关键词搜索。
- 支持随记、待办、书影音类型筛选。
- 支持进行中、计划中、已完成状态筛选。
- 搜索框使用标准放大镜图标。
- 顶部“新建”改为底部右侧绿色圆形 `+` 控件。
- 点击条目先显示预览弹窗，不直接切换页面。
- 弹窗展示类型、状态、正文、备注和记录时间。
- 动漫、韩剧的集数在弹窗右侧放大显示，并可直接 `+1`。

### 详情与编辑

- 新建和编辑统一支持类型、正文、补充说明和记录时间。
- 动漫、韩剧支持当前集数与总集数。
- 详情页的编辑、完成、进度和删除操作移至底部悬浮控件。
- 操作菜单与右侧圆形按钮连成一体，并从右向左展开。
- 收起状态显示向下箭头，展开状态显示向左箭头。
- 所有原生按钮统一使用 Flex 水平、垂直居中。

### 导航与视觉

- 页面背景恢复为暖米色 `#F3EFE4`。
- 主强调色使用绿色，减少多色分类装饰。
- 三个主页面顶部统一显示品牌名“拾羽”。
- 底部导航使用羽毛、列表、人物三个线性图标。
- 底部导航选中状态使用主题绿色，未选中状态使用灰色。

### 原型数据

- 建立统一 `PrototypeEntry` 模型，覆盖随记、待办和书影音。
- 本地数据支持新增、编辑、完成、进度更新和删除。
- 增加旧版 `phase-one-prototype-record` 数据迁移。
- 使用删除 ID 列表避免被删除的示例或迁移记录重新出现。
- 当前权威数据仍为本地 Storage，不会写入云托管 MySQL。

### 兼容性与问题修复

- 移除生成代码中的可选链 `?.` 和空值合并 `??`，修复微信预览压缩器 `Unexpected token: punc (.)`。
- 修复微信原生 `button` 默认样式将圆形按钮撑成长椭圆的问题。
- 修复首页进度 `+1` 触发动态布局重算导致白框闪烁的问题。
- 修复“更多选项”丢失首页草稿和保存后重复草稿的问题。
- 修复书籍、电影错误显示集数与剧集进度的问题。
- 修复完成的书影音被当作待办添加删除线的问题。
- 修复二级页面被直接打开时无法返回首页的问题。

### 验证方式

- 使用微信开发者工具内置 TypeScript 编译器执行严格类型检查。
- 使用微信开发者工具 `wcc` 编译器检查 WXML。
- 解析所有项目 JSON，并检查 `app.json` 注册页面的四类文件是否齐全。
- 使用 `git diff --check` 检查空白和补丁格式问题。

### 后续范围

- 用户主动选择头像、填写昵称。
- Flask 记录 API 和 MySQL 正式数据表。
- 跨设备同步、账号注销和云端数据删除。
- 真机测试、隐私配置与微信审核。
