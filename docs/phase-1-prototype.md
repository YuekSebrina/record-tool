# Phase 1 产品原型与视觉规范

## 目标

“留痕”是一款打开就能记录的私人书影音手账。Phase 1 用可运行的小程序页面确认信息架构、核心路径和视觉语言，不连接生产记录接口。

## 页面地图

| 页面 | 路由 | 作用 |
| --- | --- | --- |
| 记录首页 | `pages/index/index` | 快速录入入口、接着看、最近记录 |
| 快速录入 | `pages/add/add` | 名称、分区、状态和时间的单屏录入 |
| 全部记录 | `pages/records/records` | 搜索、分区筛选、状态筛选和记录索引 |
| 记录详情 | `pages/detail/detail` | 当前状态、进度操作、备注和时间轨迹 |
| 我的 | `pages/profile/profile` | 微信资料入口、统计、偏好和隐私设置 |

主导航固定为“记录 / 全部 / 我的”。快速录入与详情属于二级页面，不占用主导航位置。

## 核心路径

### 第一次记录

```text
打开首页
→ 点击“记录一个作品”
→ 输入名称
→ 选择分区和状态
→ 时间默认现在
→ 保存
→ 查看记录详情
```

名称是唯一必须手动输入的内容。动漫、韩剧、书籍默认选择“进行中”，电影默认选择“已看”。

### 更新剧集进度

```text
打开首页
→ 找到“接着看”条目
→ 点击 +1
→ 当前集数与进度条立即更新
```

详情页保留同样的 `+1` 操作，并提供精确编辑和标记完成入口。

### 查找已有记录

```text
进入“全部”
→ 输入名称或选择分区
→ 选择想看、进行中或已完成
→ 打开记录详情
```

无搜索结果时同时提供“清除筛选”和“新建记录”。

## 视觉语言

视觉关键词：私人手账、书籍索引、旧电影票根、编辑部档案。

| Token | 色值 | 用途 |
| --- | --- | --- |
| Paper | `#F3EFE4` | 页面底色 |
| Paper Light | `#FFFDF7` | 表单、卡片和浮层 |
| Ink | `#17211B` | 主文字、边框和主按钮 |
| Accent | `#D96941` | 关键动作、进度和强调信息 |
| Muted | `#778078` | 辅助文字 |
| Book | `#48745A` | 书籍分区 |
| Anime | `#D96941` | 动漫分区 |
| Movie | `#315E78` | 电影分区 |
| K-drama | `#9A5263` | 韩剧分区 |

标题使用系统可用的衬线字体回退组合，正文使用系统无衬线字体。数字、期号和索引使用 Georgia 风格，形成“档案编号”特征。

## 尺寸与间距

| 项目 | 规则 |
| --- | --- |
| 页面水平边距 | `36rpx` |
| 大区块间距 | `52rpx` 至 `72rpx` |
| 列表行最小高度 | `104rpx` |
| 常规点击区域 | 不小于 `64rpx` |
| 主按钮高度 | `88rpx` 至 `96rpx` |
| 常规圆角 | `3rpx` 至 `8rpx`，避免通用大圆角卡片风格 |
| 移动端底部安全区 | 使用 `env(safe-area-inset-bottom)` |
| 宽屏预览 | 内容最大宽度 `700px` 并居中 |

## 组件规则

- 主操作使用深色实心按钮，橙色只强调新增、进度和当前状态。
- 分类使用“书 / 漫 / 影 / 韩”文字印章，不依赖外部图片资源。
- 列表优先显示名称、状态、进度和时间，不使用大面积海报墙。
- 状态不能只靠颜色表达，必须同时显示文字。
- 保存前不展示评分、标签、长评等次要字段。
- 空状态必须说明原因，并提供一个直接可执行的下一步。
- 破坏性操作需要确认；正式版本删除成功后还要提供撤销。

## 原型数据边界

- 页面内置四条展示数据，用于确认排版和交互。
- 快速录入和进度更新只写入本地键 `phase-one-prototype-record`。
- 原型不会调用 `/api/count`，不会修改云托管 MySQL。
- 微信头像昵称、正式登录、跨设备同步和真实统计留到后续 Phase 实现。

## Phase 1 验收

- 首页可以在一个录入页面内完成第一条记录。
- 首页的进行中条目可以一次点击更新一集。
- 五个页面之间导航完整，无死路。
- 搜索无结果时能看到完整空状态。
- 状态同时用文字与视觉样式表达。
- 底部内容避开设备安全区。
- 手机窄屏与开发工具宽屏预览均保持可读。
