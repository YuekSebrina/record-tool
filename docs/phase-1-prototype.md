# Phase 1 产品原型与视觉规范

## 产品定位

“拾羽”是一个简单、私人的通用记录工具，不是专门的书影音管理应用。

用户打开小程序后，可以立即记录三类内容：

| 类型 | 最少填写 | 可选能力 |
| --- | --- | --- |
| 随记 | 一段文字 | 补充说明、修改记录时间 |
| 待办 | 一件要做的事 | 完成状态、补充说明、时间 |
| 书影音 | 作品名称 | 书籍、动漫、电影、韩剧分类，状态和集数 |

书影音只是通用记录的一种模板，不是首页和产品结构的中心。

## 页面结构

| 页面 | 路由 | 作用 |
| --- | --- | --- |
| 记录 | `pages/index/index` | 直接输入并保存，查看最近记录 |
| 新建/编辑 | `pages/add/add` | 填写补充说明、时间和书影音进度 |
| 全部 | `pages/records/records` | 搜索并按类型、状态筛选 |
| 详情 | `pages/detail/detail` | 查看、完成、更新、编辑和删除 |
| 我的 | `pages/profile/profile` | 微信资料入口、数量概览和设置 |

主导航保持为“记录 / 全部 / 我的”。

## 核心路径

### 记录一段话

```text
打开首页
→ 默认选择“随记”
→ 输入文字
→ 保存
```

### 添加待办

```text
打开首页
→ 选择“待办”
→ 输入事项
→ 保存
→ 在列表中直接勾选完成
```

### 记录书影音

```text
打开首页
→ 选择“书影音”
→ 输入名称
→ 选择分类和状态
→ 保存
```

集数、补充说明和自定义记录时间属于可选信息，通过“更多选项”填写。

## 简约视觉规范

| 项目 | 规则 |
| --- | --- |
| 页面背景 | `#F3EFE4` |
| 内容背景 | `#FFFFFF` |
| 主文字 | `#202420` |
| 辅助文字 | `#7A817B` |
| 主强调色 | `#3B7658` |
| 分隔线 | `#E3E7E2` |
| 页面边距 | `32rpx` |
| 常规圆角 | `8rpx` 至 `16rpx` |

界面不再使用大字号衬线标题、票根、期刊编号、旋转卡片、粗重阴影或多种分类色。信息层级只通过字号、留白、浅色边框和单一绿色强调色表达。

## 交互规则

- 首页输入框默认可直接输入，不强制进入新页面。
- 名称或正文是唯一必填内容。
- 时间默认当前时间。
- 次要字段默认隐藏在“更多选项”中。
- 待办可以在列表和详情中切换完成状态。
- 进行中的书影音可以一键更新进度。
- 删除必须二次确认。
- 所有状态同时显示文字，不只依赖颜色。
- 搜索不到内容时提供清除筛选入口。

## 原型数据边界

- 内置少量随记、待办和书影音数据用于确认排版。
- 新增、编辑、完成、进度和删除操作只保存在本地。
- 原型不会调用生产后端，也不会修改云托管 MySQL。
- 微信登录、头像昵称、跨设备同步和正式数据库留到后续 Phase。

## Phase 1 验收

- 打开首页后可以直接写下一段话并保存。
- 可以创建待办并勾选完成。
- 可以记录书籍、动漫、电影和韩剧。
- 书影音不会盖过随记和待办的主要入口。
- 五个页面之间没有导航死路。
- 界面保持单一强调色、适量留白和紧凑列表。
- 手机窄屏与开发者工具宽屏预览均保持可读。
